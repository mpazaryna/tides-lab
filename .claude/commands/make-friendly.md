# Make Code Junior-Developer Friendly

Add inline comments to help a junior developer navigate and understand the codebase.

## Goal

Reduce cognitive load by labeling high-level sections, patterns, and functionality.

## Requirements

- **Label** all important sections and functionality
- **Friendly and engaging** tone
- **Minimal words** - don't overexplain
- **Descriptions** only for complex interdependencies
- Focus on **high-level components and patterns**

## Target

Junior developer doing code review and modifications.
